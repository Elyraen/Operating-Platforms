# Operating Platforms

Projects completed as part of my Operating Platforms course at Southern New Hampshire University (SNHU).

## Overview

This repository showcases my work in designing and analyzing software systems across different operating platforms. The primary focus of these projects is understanding how applications are developed, deployed, and scaled in both traditional and cloud-based environments.

## Featured Project: Draw It or Lose It

### Client Summary
The client, The Gaming Room, requested a redesign of their existing Android-based game *Draw It or Lose It* into a web-based application. The goal was to support multiple users, teams, and game sessions across various platforms while maintaining performance, scalability, and reliability.

### Key Requirements
- Develop a web-based application using object-oriented programming principles  
- Support multiple teams and players with unique identifiers  
- Ensure cross-platform compatibility (desktop and mobile)  
- Implement a scalable and maintainable architecture  
- Use a Singleton design pattern for efficient memory and instance management  

## What I Did Well

I effectively designed a clear system architecture and demonstrated strong use of object-oriented concepts such as inheritance, encapsulation, and modular design. I also connected technical decisions—like using Linux and a client-server model—to real-world application needs.

## What I Learned

Working through the design document helped me better understand how to plan software before development. It improved my ability to:
- Break down complex systems into manageable components  
- Design scalable and distributed applications  
- Think critically about system requirements and constraints  

## Areas for Improvement

If I were to revise this project, I would improve the depth of analysis in platform comparisons and provide more detailed explanations of trade-offs such as cost, performance, and scalability.

## Design Approach

I approached the design by:
- Applying object-oriented principles (inheritance, encapsulation, abstraction)  
- Structuring the system into logical components (Game, Team, Player, GameService)  
- Considering scalability and performance in a distributed environment  
- Aligning technical decisions with user and business requirements  

## Technologies & Concepts

- Object-Oriented Programming (OOP)  
- Client-Server Architecture  
- Cloud Computing Concepts  
- Linux-Based Systems  
- Distributed Systems  
- RESTful APIs  
- Containerization (Docker)  

## Why User Needs Matter

Understanding user requirements is critical because it ensures the system is not only functional but also usable and scalable. Designing with the user in mind leads to better performance, accessibility, and long-term success of the application.
